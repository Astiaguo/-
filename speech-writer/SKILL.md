# Speech Writer — 演讲稿生成

将材料转化为引人入胜的演说辞，同时精准克隆演讲者的个人语言风格。

## Trigger

用户提供以下材料：
1. **【核心内容】**：需要讲解的材料（PDF、PPT 等），包含要讲的核心论点、数据和逻辑
2. **【历史参考】**：演讲者以往的演讲稿或发言材料（Word、Markdown、HTML 格式均可）

**用户可能一次性提供全部材料，也可能分批给出。** 接收完毕后开始生成。

## 工作流程

### Step 1: 风格分析
仔细分析【历史参考】中的语言特征，提取以下维度：
1. **句式长短**：偏长句还是短句为主？
2. **幽默感密度**：幽默是密集还是克制？用什么方式幽默（自嘲、反讽、类比、谐音梗等）？
3. **专业术语习惯**：是否频繁使用行业术语？还是偏通俗表达？
4. **节奏感**：语速快慢的暗示、停顿的使用、高潮铺垫方式
5. **口头禅/标志性表达**：是否有反复出现的句式或语气词？
6. **互动方式**：如何与听众互动（提问、留白、引导掌声等）？

### Step 2: 内容理解
通读【核心内容】，提取：
- 核心论点
- 关键数据和事实
- 逻辑框架和章节结构
- 需要强调的重点

### Step 3: 逐页生成
按 PDF/PPT 的页码逐页输出演讲稿内容。

## 输出格式

严格按以下格式输出：

```
--- P[页码] ---

[演讲稿正文]
```

每页独立输出，清晰标注页码。

## 核心原则

1. **100% 风格克隆**：新演讲稿的语言风格必须与【历史参考】高度一致，读者/听众能听出是同一人在说话。
2. **内容忠实度**：核心论点、数据和逻辑必须完全基于【核心内容】，**不得凭空捏造事实或数据**。
3. **口语化表达**：文字必须"适合朗读"。
   - 多用短句，避免过长的从句
   - 避免书面化的复杂句式
   - 避免"接下来这招叫："等武林化表达
4. **逐页对应**：每页的演讲稿内容必须与该页的核心信息匹配，不能张冠李戴。

## 节奏感处理

演讲稿的节奏感是区别于普通文章的关键：
- **开场**：吸引注意力，建立与听众的联系
- **过渡**：自然承接上一页/上一段，不要生硬跳转
- **高潮**：在关键数据/观点处用短句或停顿制造冲击
- **收尾**：每页结尾留有悬念或小结，引导听众关注下一页

## 输入格式说明

用户可以用任何方式提供材料：
- **核心内容**：PDF / PPT 文件（AI 需要能读取文件内容）
- **历史参考**：Word / Markdown / HTML 文件，或直接在对话中粘贴文本
- 用户也可以上传多个参考文档，AI 需要综合分析风格

## 交互方式

1. 如果用户一次性给了所有材料 → 直接开始生成，建议先输出前 N 页（如 P1-P20），用户确认满意后再继续
2. 如果用户分批给材料 → 等材料收齐后再开始
3. 如果用户对某页不满意 → 可以针对性调整那一页的风格或内容

## 注意事项

- 不要添加原文中没有的数据或论点
- 风格模仿是语言层面的，不是内容层面的——内容必须忠于原材料
- 口语化不等于随意化——专业场合的演讲需要保持专业度，只是表达方式更"能读"
- 如果核心材料中没有某页的具体内容，标注"此页无核心内容可讲"或向用户确认

## Example Output

```
--- P1 ---

大家好，我是[演讲者名字]。今天想和大家聊一个我们团队想了很久、也做了很久的话题。

先问大家一个问题——[结合P1内容的互动开场]

--- P2 ---

回到刚才的问题。P2 上这组数据，说实话，第一次看到的时候我也愣了一下。

[基于P2内容的具体讲解，用短句，保持节奏感]

--- P3 ---

所以接下来我们做了什么呢？

[基于P3内容的讲解]
```
