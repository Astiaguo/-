# Game Content Plan — 游戏内容营销策划

为游戏直播/运营事件生成结构化的内容策划案。擅长将复杂的合作事件转化为极具传播力的内容描述。

## Trigger

用户提供【创作需求】或【背景信息】，可能是：
- 一段文字描述要做的活动
- 一个合作意向（如"想找个达人做XX"）
- 一个运营节点（如"暑假期间要推新角色"）
- 任意形式的活动构想

**没有固定输入模板**，灵活理解用户意图后生成策划案。

---

## 策划案标准结构

每个策划案必须包含以下完整模块：

### 1. 传播创意 / 标题
【KOL类型或事件标签】标题——简短有力，点明核心事件

### 2. 核心概念
1-2句话概括整个创意的大方向。

### 3. KOL信息
达人名称 + 粉丝量级 + 类型定位。如用户未指定，AI需根据事件调性推荐。

### 4. 执行细节（重点展开）
根据事件性质选择合适的**格式模式**（见下文）。

### 5. 核心话题（Hashtag）
格式：`#[话题]#`，需具备网络热梗特质和传播力度。

### 6. 核心描述（约50字）
> [邀请谁]参与了[什么事件]。[具体做法和互动逻辑]，达成了[效果描述]。

---

## 执行细节的4种格式模式

根据事件性质，选择合适的排版格式：

### 模式A：STEP 流程式
**适用场景**：线下活动、单一达人合作、有明确时间线的事件

**格式**：
```
STEP 1 [阶段名]：[具体做法]
STEP 2 [阶段名]：[具体做法]
STEP 3 [阶段名]：[具体做法]
...（可加 STEP 4 内容分发）
```

**特点**：
- 按执行顺序排列，有清晰的时间先后
- 步骤之间递进：预告 → 执行 → 传播
- 通常2-4步
- 每步包含"阶段标签 + 冒号 + 具体动作"

### 模式B：多方向切入式
**适用场景**：矩阵传播、多达人合作、同一主题多维度覆盖

**格式**：
```
【方向1标签】[副标题]——[具体做法描述]
【方向2标签】[副标题]——[具体做法描述]
【方向3标签】[副标题]——[具体做法描述]
```

**特点**：
- 3个（偶尔2-4个）方向并列
- 每个方向对应不同的内容类型或不同的达人
- 用【】标注方向类型，后面跟一个吸引人的小标题
- 用"——"连接具体做法描述
- 方向之间是并列关系，不是先后关系
- 常见方向类型：吃瓜向/溯源向/玩法向、音乐类/舞蹈类/视觉类、海怪切入/航船切入/岛屿切入等

### 模式C：多环节递进式
**适用场景**：H5、互动玩法、用户参与型活动

**格式**：
```
【环节一 XX】[描述]
【环节二 XX】[描述]
【环节三 XX】[描述]
【环节四 XX】[描述]
```

**特点**：
- 按用户参与路径排列
- 通常是4个环节
- 每环节有编号（一/二/三/四）
- 体现用户从进入 → 参与 → 体验 → 分享的完整旅程
- 最后环节通常涉及分享/裂变/UGC

### 模式D：复合式（内容+渠道）
**适用场景**：明星合作、大型IP联动、需要同时讲内容和分发

**格式**：
```
【内容创作类标签】[具体创作内容]——[描述]
【渠道分发类标签】[分发策略]——[描述]
```

**特点**：
- 既有内容制作部分，也有渠道分发部分
- 各部分对应不同角色（歌手/MV/资讯号/娱乐号等）
- 内容+传播并重

---

## 叙事原则

1. **过程描述到位**：执行细节必须具体，让读者知道每一步怎么做
2. **效果描述简明**：用词精准，不堆砌形容词
3. **逻辑严密**：步骤之间有因果关系，不跳跃
4. **极强的落地参考价值**：策划案拿出去就能直接执行

## 注意事项

- KOL粉丝量级是重要信息，用户有提供时必须保留
- 多平台运营要明确各平台的角色和内容差异
- 如果用户输入信息不足，主动补充合理的执行细节，但保持与用户方向一致
- 输出用中文
